# איך לפנות אליי ב-Claude Code

מדריך כיס לשיחות עתידיות על פרוייקט JobSwipe.

## פקודות מהירות (slash commands)

| פקודה | מה היא עושה |
|---|---|
| `/status` | סיכום מהיר של מצב הפרוייקט + build + 3 המלצות מה הלאה |
| `/demo` | משתמשי דמו + 3 תרחישי בדיקה עם URLs מדויקים לדפדפן |
| `/smoke` | מפעיל dev server ודופק curl על כל המסלולים הקריטיים |
| `/next` | דירוג 3 המשימות הבאות הכי חשובות להשקה (מתכנן בלי לממש) |

את הפקודות האלה מקלידים בתחילת הודעה — למשל פותחים שיחה חדשה וכותבים `/status` ושולחים.

## ניסוח חופשי (בלי slash)

גם אלה עובדים — הסקילים מזהים את ההקשר ונטענים לבד:

- "מה הסטטוס?" / "מה המצב?" → טוען orientation
- "איך אני רואה את התוצר?" / "תראה לי איך להריץ" → טוען ops
- "תוסיף X", "תתקן Y", "תשנה Z" → טוען את הסקילים הרלוונטיים לפי הקבצים שצריך לגעת בהם
- "תרוץ build" / "סמוק טסט" → טוען ops ומריץ

## בחינה ידנית בדפדפן (בלעדיי)

```bash
cd jobswipe
npm run dev
# http://localhost:3000
# משתמש לדוגמה: pizzaria@demo.com  סיסמה: demo1234
```

רשימת משתמשי דמו המלאה מופיעה ב-`README.md` או בפלט של `/demo`.

## מה קיים תחת `.claude/`

- `skills/` — 4 סקילים שמרכזים ידע על הפרוייקט: `jobswipe-orientation`, `jobswipe-nextjs16`, `jobswipe-data-model`, `jobswipe-ops`.
- `commands/` — 4 slash commands (הקבצים שמפעילים את הפקודות למעלה).

שינוי של אחד מהקבצים האלה = שינוי התנהגות שלי בשיחות עתידיות. אפשר לבקש ממני לעדכן אותם בכל שלב.
