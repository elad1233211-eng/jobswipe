# JobSwipe — גיוס בסוויפ 💼❤️

MVP של פלטפורמת גיוס מהיר בסגנון Tinder, בעברית (RTL), ממוקדת בעבודות "צווארון כחול" בישראל (מלצרות, שליחויות, קופאים, ניקיון, מחסן, בנייה וכו').

> **סטטוס:** MVP לפיתוח מקומי. אין שירותים חיצוניים בתשלום. הכל רץ מקומית על SQLite.

---

## מה יש כאן

- 🧑‍💼 **שני סוגי משתמשים**: מועמד/ת או מעסיק/ה
- 📝 **אונבורדינג מהיר**: מינימום שדות, emoji כאווטר במקום העלאת תמונות
- 🔥 **Feed סוויפ** למועמדים — גלישה על משרות פעילות עם אנימציה של גרירה
- 🎯 **Feed מעסיק** — המעסיק רואה את המועמדים שכבר עשו לייק על אחת המשרות שלו
- 💘 **מנגנון Match דו-צדדי**: שני הצדדים חייבים לייק כדי ליצור התאמה
- 💬 **צ'אט פנימי** בין הצדדים שנוצרה ביניהם התאמה
- 🧭 **ניווט תחתון** במובייל ראשון (עיצוב רספונסיבי מלא)

## Stack

- **Next.js 16.2** (App Router, Turbopack, React 19.2)
- **TypeScript**
- **Tailwind CSS 4**
- **better-sqlite3** — DB מקומית, אפס קונפיג (קובץ `data/jobswipe.db`)
- **framer-motion** — אנימציית הסוויפ
- **jose + bcryptjs** — JWT בעוגייה httpOnly
- **zod** — ולידציית קלט

## הרצה מקומית

```bash
# מהתיקייה jobswipe/
npm install          # כבר הורץ
npm run seed         # טעינת נתוני דמו (מעסיקים, משרות, מועמדים)
npm run dev          # שרת פיתוח על http://localhost:3000
```

### תרחישי בדיקה מהירים

1. **Match מלא**: התחבר כ-`pizzaria@demo.com` → כנס למשרה "מלצר/ית לסופי שבוע" → תראה את נועה מחכה → סמן לייק → מופיעה התאמה ומייד אפשר לשלוח הודעה.
2. **סוויפ טרי**: התחבר כ-`maya@demo.com` → המסך הראשי הוא ה-feed של המשרות. גרור כרטיסים שמאלה/ימינה או השתמש בכפתורים.
3. **פרסום משרה**: התחבר כמעסיק → לחץ "+ פרסום משרה חדשה".
4. **הרשמה מאפס**: `/signup` — בחר מועמד או מעסיק, עבור onboarding, תגיע ל-feed/דשבורד.

│   ├── layout.tsx            # RTL + עברית
│   └── page.tsx              # Landing
├── lib/
│   ├── db.ts                 # סכמת SQLite + טיפוסים

## מודל הנתונים

- `users` — (id, email, password_hash, role)
- `candidates` / `employers` — פרופילים מורחבים (FK ל-users)
- `jobs` — משרות של מעסיק, קטגוריה, עיר, שכר, דרישות (JSON)
- `swipes` — (from_user_id, target_kind, target_id, direction) — כל סוויפ שמישהו עשה על משרה או על מועמד
- `matches` — נוצר אוטומטית רק כששני הצדדים עשו לייק על אותה משרה
- `messages` — הודעות צ'אט תוך match

## החלטות עיצוב שהתקבלו ב-MVP

- **Web-first PWA-like**: אפליקציית Next.js רספונסיבית (לא React Native) — מתפשרת על native feel אבל מאפשרת משלוח מהיר לבדיקה מול משתמשים אמיתיים. מעבר ל-React Native אפשרי בהמשך, אותו backend.
- **Emoji כאווטר**: אין העלאת קבצים ב-MVP. נמנע מאחסון S3/R2 ומבעיות מודרציה של תוכן ויזואלי.
- **התאמה דו-צדדית מלאה**: מונעת הטרדה מצד מעסיקים — מעסיק רואה רק מועמדים שסימנו לייק על המשרה שלו.
- **פילטר אזור/קטגוריה**: *עדיין לא* ב-feed — יתווסף כשנכניס יותר משרות. כרגע כל המשרות הפעילות נטענות ממוינות לפי תאריך.
- **אין תמונות מוצר, אין וידאו, אין AI matching** — כולם פיצ'רים פוסט-MVP.

## פריסה

פריסה מלאה מופיעה ב-[`DEPLOYMENT.md`](./DEPLOYMENT.md). תמצית:

```bash
# על שרת חדש עם Docker:
echo "AUTH_SECRET=$(node -e 'console.log(require(\"crypto\").randomBytes(48).toString(\"base64url\"))')" > .env
docker compose up -d --build
# בריאות:
curl http://localhost:3000/api/health
```

## אזהרות להשקה לפרודקשן

- **החלף את `AUTH_SECRET`** — ראה `.env.local.example`. לייצור, השתמש במחרוזת אקראית של 32+ תווים.
- **SQLite ≠ Production DB אמיתי לקנה מידה** — מצוין ל-MVP ול-50-100 משתמשים במקביל. ברגע שמתחילים לגייס בטא-יוזרים, שקול לעבור ל-Postgres (קל, אין הבדלי SQL גדולים).
- **מודרציה** — יש פילטר בסיסי של קללות ו-rate limiting, אבל לא מחליף בדיקה אנושית על דיווחים.
- **GDPR/הגנת פרטיות** — דפי `/legal/privacy`, `/legal/terms`, `/legal/accessibility` קיימים אבל צריך לעדכן את פרטי הישות המשפטית (שם, דוא"ל, כתובת) לפני השקה.
- **אפליה**: אל תוסיפו בעתיד פילטרים לפי גיל/מגדר/מוצא כפילטרים של מעסיק — זה עלול להוות הפרת חוק שוויון הזדמנויות בעבודה.

## למה דווקא ככה

**מה שחשוב ב-MVP זה לא הקוד — זה האם יש product-market fit.** הסטק המקומי הזה נותן לך להריץ לפני הפנים של מעסיקים ומועמדים אמיתיים ביום אחד, ולאסוף feedback. ברגע שיש הוכחת קונספט — קל להחליף כל רכיב (DB, hosting, mobile wrapper) בלי לשבור ארכיטקטורה.
